2.1 Inserting Data (INSERT INTO)

The INSERT INTO statement is used to add new rows of data into a table.

Syntax 1: Specifying column names and values:


INSERT INTO table_name (column1, column2, column3, ...)
VALUES (value1, value2, value3, ...);

The order of values must match the order of the specified columns. This is the recommended syntax as it's explicit and less prone to errors if the table structure changes.

Syntax 2: Adding values for all columns (order matters):


INSERT INTO table_name
VALUES (value_for_col1, value_for_col2, value_for_col3, ...);

Here, you must provide a value for every column in the table in the exact order they appear in the table's definition. If a column allows NULL or has a default value (like an auto-incrementing ID), you might still need to provide a placeholder (e.g., `NULL`).

Example: Add a new user 'Frank Furter' to the Marketing department (ID 3):


INSERT INTO users (name, email, signup_date, department_id)
VALUES ('Frank Furter', 'frank@example.com', '2024-04-01', 3);

Note: We don't specify the `id` column because it's set to `AUTOINCREMENT` in our schema (handled automatically by SQLite/sql.js in this setup).

You can insert multiple rows at once:


INSERT INTO users (name, email, department_id)
VALUES
    ('Grace Hopper', 'grace@example.com', 2),
    ('Henry Ford', 'henry@example.com', 2);
